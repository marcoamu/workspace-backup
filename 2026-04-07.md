
## Fix Importante: Prompt Enhancer con MiniMax (15:16)

### Problema:
El Prompt Enhancer no estaba usando MiniMax correctamente - hacía fallback a template.

### Causa raíz:
1. Endpoint incorrecto: `api.minimax.chat` en vez de `api.minimax.io/anthropic/v1/messages`
2. API key no se cargaba desde .env (no había load_dotenv)
3. Response parsing incorrecto (buscaba `choices[0].message.content` en vez de `content[].text`)

### Solución aplicada:
1. ✅ Endpoint: `https://api.minimax.io/anthropic/v1/messages`
2. ✅ Modelo: `MiniMax-M2.7`
3. ✅ Headers: `anthropic-version: 2023-06-01`
4. ✅ Response parsing: iterar sobre `content` array buscando `type: "text"`
5. ✅ API key guardada en `/mibimax-app/backend/.env`
6. ✅ `from dotenv import load_dotenv` + `load_dotenv()` al inicio de server.py

### Archivos modificados:
- `/mibimax-app/backend/server.py`
- `/mibimax-app/backend/.env` (creado)

### NO MODIFICAR:
- La lógica de llamada a MiniMax en enhance_prompt_smart()
- El parsing del response de MiniMax
- La carga de .env al inicio
